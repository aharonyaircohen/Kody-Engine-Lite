## Git utilities (`src/git-utils.ts`)

All use `execFileSync` with hook-safe env (`HUSKY=0`, `SKIP_HOOKS=1`, cached).

| Function | Purpose | Details |
|----------|---------|---------|
| `deriveBranchName(num, title)` | Generate branch name | Slugify, limit 50 chars, format: `<num>-<slug>` |
| `getDefaultBranch(cwd?)` | Detect default branch | Three-fallback: symbolic-ref â†’ remote show (10s) â†’ `"dev"` |
| `ensureFeatureBranch(num, title)` | Create/checkout branch | Check if already on feature branch, fetch origin, check remote/local, create with tracking. `git clean -fd` in CI. |
| `getCurrentBranch()` | Current branch name | `git branch --show-current` |
| `commitAll(message)` | Stage + commit | Check `git status --porcelain` first, `--no-gpg-sign`, return `{ success, hash, message }` |
| `pushBranch()` | Push to origin | `git push -u origin HEAD`, 120s timeout |
| `getChangedFiles(base)` | Diff file list | `git diff --name-only origin/<base>...HEAD` |
| `getDiff(base)` | Full diff text | `git diff origin/<base>...HEAD`, empty string on error |

## GitHub API (`src/github-api.ts`)

All use `gh` CLI via `execFileSync` with 30s timeout. Auth: `GH_PAT` (preferred) â†’ `GH_TOKEN` fallback.

| Function | Purpose |
|----------|---------|
| `getIssue(num)` | Fetch body + title (`gh issue view --json`) |
| `setLabel(num, label)` | Add label (`gh issue edit --add-label`) |
| `removeLabel(num, label)` | Remove label |
| `postComment(num, body)` | Comment via stdin (`--body-file -`) |
| `createPR(head, base, title, body)` | Create PR, extract number from URL |
| `setLifecycleLabel(num, phase)` | Atomic swap: remove all `kody:*` labels, add new one |
| `closeIssue(num, reason?)` | Close with reason (default: "completed") |

Lifecycle labels: `["planning", "building", "review", "done", "failed"]`

## State machine updates

### Git integration points
- **Pipeline start:** `ensureFeatureBranch(issueNumber, taskTitle)` if issueNumber provided
- **After build (completed):** `commitAll("feat(<taskId>): implement task")`
- **After review-fix (completed):** `commitAll("fix(<taskId>): address review")`

### Ship stage (replaces stub)
1. `getCurrentBranch()` â†’ head
2. `getDefaultBranch()` â†’ base
3. `pushBranch()` with hook-safe env
4. Resolve owner/repo: `config.github` or parse `git remote get-url origin` (`/github\.com[/:]([^/]+)\/([^/.]+)/`)
5. Derive PR title from task.md (first non-heading line, max 72 chars)
6. `createPR(head, base, title, body)` â€” body: "Generated by Kody pipeline"
7. `postComment(issueNumber, "đźŽ‰ PR: <url>")` â€” fire-and-forget
8. Write `ship.md` with PR URL

### Lifecycle labels (set at transitions)
- Pipeline start: `kody:planning` (or `kody:building` if rerun)
- Enter build: `kody:building`
- Enter review: `kody:review`
- Completed: `kody:done`
- Failed/timeout: `kody:failed`

### Local mode
When `ctx.input.local === true`: skip ALL GitHub API calls (labels, comments, PR). Still do git ops.

## GitHub Actions workflow

### Triggers
```yaml
on:
  workflow_dispatch:
    inputs:
      task_id: { required: true, type: string }
      mode: { type: string, default: "full" }
      from_stage: { type: string, default: "" }
      issue_number: { type: string, default: "" }
      feedback: { type: string, default: "" }
      dry_run: { type: boolean, default: false }
      runner: { type: choice, options: [github-hosted, self-hosted], default: github-hosted }
      use_test_config: { type: boolean, default: false }
  issue_comment:
    types: [created]
  pull_request_review:
    types: [submitted]
  push:
    branches: [main, dev]
    paths: [src/**, prompts/**, package.json]
```

### Jobs

**1. Parse** (issue_comment trigger only)
- Sparse checkout `src/ci/` only
- Run `parse-safety.ts` â†’ validate author
- Add "eyes" emoji reaction
- Run `parse-inputs.ts` â†’ extract inputs
- Output variables

**2. Orchestrate** (main job, 120min timeout)
- GitHub App token generation (`actions/create-github-app-token`)
- Checkout (fetch-depth: 0, persist-credentials, App token)
- Setup pnpm + Node 22
- Install Claude Code CLI
- Configure git identity
- Set API keys from secrets
- Run `pnpm kody` with env vars
- Write pipeline summary to `$GITHUB_STEP_SUMMARY`
- Upload `.tasks/<id>/` artifacts (7-day retention)

**3. notify-parse-error** â€” post usage message on bad `@kody` commands
**4. notify-orchestrate-error** â€” post failure with logs link, set `kody:failed`
**5. smoke-test** (push trigger) â€” typecheck, CLI validation, dry-run
