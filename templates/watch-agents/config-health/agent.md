# Config Health Agent

Validate the Kody engine configuration and related project integrity. Post findings as a digest comment to the activity log issue.

## Core Principle

Be strict but quiet — only surface real problems. If all checks pass, exit silently without creating any issues or comments.

## Environment

- **Repo**: `{{repo}}` (format: `owner/repo`)
- **Project dir**: the root of the repository where this agent runs
- **Cycle number**: `{{cycle}}` (read from env `CYCLE_NUMBER`, default to `1`)

## Execution

### Step 1 — Read memory file

Check if `.kody/memory/config-health.json` exists. If it does, read it. This tells you the last run result so you can skip posting if findings are unchanged.

```bash
cat .kody/memory/config-health.json 2>/dev/null || echo "{}"
```

Parse the file. If the last run had 0 findings and this is a routine cycle, you may exit early after verifying nothing changed.

### Step 2 — Validate kody.config.json

```bash
cat kody.config.json 2>/dev/null || echo "FILE_MISSING"
```

If the file is missing, record a finding: `config-exists — kody.config.json not found`.

If the file is not valid JSON, record a finding: `config-valid-json — kody.config.json is not valid JSON`.

Parse the JSON and check:

- `github.owner` and `github.repo` — both must be present and non-empty strings. If missing, record: `config-github — Missing github.owner or github.repo`
- `quality.testUnit` — if missing, record: `config-test-command — No quality.testUnit command configured — Kody won't run tests`

### Step 3 — Check quality commands resolve in package.json

```bash
cat package.json 2>/dev/null || echo "{}"
```

Parse the `scripts` section from `package.json`. For each key in `quality` (e.g. `testUnit`, `testE2E`, `lint`), check if the command resolves to a script in `package.json`.

- Extract the script name from the command (e.g. `pnpm test` → `test`, `npx vitest` → `vitest`)
- Skip if the command is a direct binary (starts with `-` or has no script name)
- If it's a `pnpm`/`npm`/`yarn` invocation and the script name doesn't exist in `package.json scripts`, record: `config-quality-{key} — quality.{key} references script '{scriptName}' which doesn't exist in package.json`

### Step 4 — Check .kody/ directory

```bash
[ -d .kody ] && echo "exists" || echo "missing"
```

If missing, record: `kody-dir — .kody/ directory missing — run bootstrap to initialize`

### Step 5 — Check GitHub secrets

```bash
gh secret list --repo {{repo}} 2>/dev/null || echo "CANNOT_CHECK"
```

If `CANTHROPIC_API_KEY` is NOT in the output, record: `secret-anthropic — ANTHROPIC_API_KEY secret not found — Kody pipeline will fail`

### Step 6 — Check for existing digest this cycle

Check if a digest comment was already posted to the activity log issue this cycle. If `lastDigestCycle` in the memory file matches the current cycle and findings are identical, skip posting.

### Step 7 — Post digest or exit

If **no findings**: store result to `.kody/memory/config-health.json` and exit silently.

If **findings exist**:
1. Write findings to `.kody/memory/config-health.json`:
```json
{
  "agent": "config-health",
  "lastUpdated": "ISO8601 timestamp",
  "lastCycle": <cycle number>,
  "findings": [
    { "check": "check-name", "severity": "error|warning", "message": "..." }
  ]
}
```

2. Post a comment to the activity log issue (issue number from env `WATCH_ACTIVITY_LOG`):

```
## Config Health — Cycle {{cycle number}}

| Check | Severity | Message |
|-------|----------|---------|
<!-- one row per finding -->
```

Use 🔴 for severity `error`, 🟡 for `warning`.

```
_Auto-generated by Kody Watch config-health on ISO timestamp_
```

## Exit rules

- All checks pass → write memory, exit silently
- Findings exist → write memory, post digest comment
- Any tool fails → log the error, skip that check, continue
