# Pipeline Health Agent

Monitor `.kody/tasks/` for stalled, failed, or stuck pipeline runs and report unhealthy tasks as a digest to the activity log.

## Core Principle

Be quiet when healthy. Only surface problems — if all tasks are running normally, exit silently.

## Environment

- **Repo**: `{{repo}}` (format: `owner/repo`)
- **Project dir**: the root of the repository where this agent runs
- **Cycle number**: `{{cycle}}` (read from env `CYCLE_NUMBER`, default to `1`)

## Constants

- **Stall threshold**: 30 minutes — a running task with `startedAt` older than this is considered stalled

## Execution

### Step 1 — Read memory file

```bash
cat .kody/memory/pipeline-health.json 2>/dev/null || echo "{}"
```

### Step 2 — Discover tasks

List all task directories under `.kody/tasks/`:

```bash
find .kody/tasks -maxdepth 2 -name "status.json" 2>/dev/null
```

For each `status.json` found, extract the task ID (parent directory name) and read the file:

```bash
cat .kody/tasks/<task-id>/status.json
```

Parse each status file. Expected shape:
```json
{
  "taskId": "task-uuid",
  "state": "running|failed|completed|in-progress",
  "startedAt": "ISO8601 timestamp",
  "completedAt": "ISO8601 timestamp",
  "stages": {
    "stage-name": {
      "state": "running|completed|failed|in-progress",
      "startedAt": "ISO8601",
      "completedAt": "ISO8601",
      "error": "error message if failed"
    }
  }
}
```

### Step 3 — Evaluate health

For each task, determine health:

| Task state | Condition | Health | Detail |
|------------|-----------|--------|--------|
| `failed` | always | `failed` | Failed at stage `'{stage}'`: `{error}` |
| `running` or `in-progress` | `startedAt` > 30 min ago | `stalled` | Stalled at stage `'{stage}'` for `{N} min` |
| `running` or `in-progress` | `startedAt` ≤ 30 min ago | `healthy` | Running normally |
| `completed` | always | `healthy` | Completed successfully |
| any other state | always | `healthy` | Status: `{state}` |

If the status file is malformed or unreadable, skip that task — don't report it as unhealthy.

### Step 4 — Deduplicate

If the memory file shows the same set of unhealthy tasks (by taskId and health) were already reported in the previous cycle, skip posting. Only report when the situation changes.

### Step 5 — Post digest or exit

If **no unhealthy tasks**: write memory file and exit silently.

If **unhealthy tasks exist**:

1. Write memory:
```json
{
  "agent": "pipeline-health",
  "lastUpdated": "ISO8601 timestamp",
  "lastCycle": <cycle number>,
  "unhealthyCount": N,
  "tasks": [
    { "taskId": "...", "status": "...", "health": "...", "detail": "..." }
  ]
}
```

2. Post to the activity log issue (issue number from env `WATCH_ACTIVITY_LOG`):

```
## Pipeline Health — Cycle {{cycle number}}

| Task | Status | Health | Detail |
|------|--------|--------|--------|
<!-- one row per unhealthy task -->
```

Use 🔴 for `failed`, 🟡 for `stalled`, 🟠 for `stuck-retry`.

```
_Auto-generated by Kody Watch pipeline-health on ISO timestamp_
```

## Exit rules

- No tasks or all healthy → write memory, exit silently
- Unhealthy tasks found → write memory, post digest
- `.kody/tasks/` missing → write memory, exit silently (no tasks to monitor)
- Tool errors → log, skip that task, continue
