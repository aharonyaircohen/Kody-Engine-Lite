# Configuration

## `kody.config.json`

Generated by `kody-engine-lite init` with auto-detected values. Full reference:

```json
{
  "quality": {
    "typecheck": "pnpm tsc --noEmit",
    "lint": "pnpm lint",
    "lintFix": "pnpm lint --fix",
    "formatFix": "pnpm prettier --write .",
    "testUnit": "pnpm vitest run"
  },
  "git": {
    "defaultBranch": "main"
  },
  "github": {
    "owner": "your-org",
    "repo": "your-repo"
  },
  "agent": {
    "provider": "anthropic",
    "modelMap": {
      "cheap": "haiku",
      "mid": "sonnet",
      "strong": "opus"
    }
  },
  "contextTiers": {
    "enabled": true,
    "tokenBudget": 8000
  }
}
```

### Quality Commands

| Field | Description | Default | Used By |
|-------|-------------|---------|---------|
| `typecheck` | TypeScript type check | `""` | verify stage |
| `lint` | Lint check | `""` | verify stage |
| `lintFix` | Auto-fix lint | `""` | autofix (before retry) |
| `formatFix` | Auto-fix format | `""` | autofix (before retry) |
| `testUnit` | Unit test command | `""` | verify stage |

Empty string = skip that check. Commands are run via `execFileSync` with a 60-second timeout.

### Git

| Field | Description | Default |
|-------|-------------|---------|
| `defaultBranch` | PR base branch | `"dev"` |

### GitHub

| Field | Description | Default |
|-------|-------------|---------|
| `owner` | GitHub organization or username | auto-detected from git remote |
| `repo` | Repository name | auto-detected from git remote |

### Agent

| Field | Description | Default |
|-------|-------------|---------|
| `provider` | LLM provider (`"anthropic"`, `"minimax"`, `"openai"`, `"google"`) | `"anthropic"` |
| `modelMap.cheap` | Model for taskify, verify diagnosis | `"haiku"` |
| `modelMap.mid` | Model for build, review-fix, autofix | `"sonnet"` |
| `modelMap.strong` | Model for plan, review | `"opus"` |

**Partial modelMap:** You can override individual tiers without specifying all three. For example, `{ "agent": { "modelMap": { "strong": "opus" } } }` keeps the defaults for cheap and mid. The config is deep-merged.
| `defaultRunner` | Default runner name (advanced) | `"claude"` |
| `stageRunners` | Per-stage runner overrides (advanced) | — |

When `provider` is set to anything other than `"anthropic"`, Kody auto-starts a LiteLLM proxy that routes Claude Code API calls to your provider. See [LiteLLM guide](LITELLM.md).

### Context Tiers

Each pipeline stage gets context injected — memory, plan, prior stage summaries. Later stages accumulate more context. Without tiers, this can consume thousands of tokens. Tiered loading summarizes context to stay within budget:

| Tier | Tokens | Content | Use case |
|------|--------|---------|----------|
| **L0** | ~100 | First heading + first sentence. JSON: key fields only | Quick reference when full content isn't needed |
| **L1** | ~300-500 | All headings + first sentence of each section + bullets | Default for most context sources |
| **L2** | Full | Untruncated content | When a stage needs full detail (e.g., build reads plan at L2) |

Each stage has a default policy per source. For example, the build stage reads the plan at L2 (full detail) but memory at L1 (overview). Cached per file to avoid reprocessing. See [Architecture — Prompt Assembly](ARCHITECTURE.md#prompt-assembly) for implementation details.

| Field | Description | Default |
|-------|-------------|---------|
| `contextTiers.enabled` | Enable L0/L1/L2 tiered context loading | `true` |
| `contextTiers.tokenBudget` | Max tokens for context injection per stage | `8000` |

## Step Files (`.kody/steps/`)

Generated by `bootstrap` (not `init`), these are the customized prompt templates for each pipeline stage. They contain the engine's default prompt plus three repo-specific sections tailored to your codebase.

| File | Stage | What bootstrap customizes |
|------|-------|---------------------|
| `taskify.md` | taskify | Scope detection based on your project structure |
| `plan.md` | plan | Planning guidelines referencing your architecture |
| `build.md` | build | Your coding patterns, anti-patterns to fix, quality checklist |
| `autofix.md` | verify (on failure) | Fix strategies for your specific toolchain |
| `review.md` | review | Review criteria calibrated to your repo's quality bar |
| `review-fix.md` | review-fix | How to address findings in your codebase |

**Customization:** Edit any step file directly — changes take effect on the next pipeline run. No engine update needed.

**Regeneration:** Run `kody-engine-lite bootstrap --force` to re-analyze your codebase and regenerate all step files. Useful after major refactors. Previous customizations will be overwritten — commit them first if you want to compare.

**How it works:** During `bootstrap`, each default prompt template from the engine is sent to an LLM along with your project context (package.json, README, CLAUDE.md, AGENTS.md, sample source files, detected architecture). The AI produces a customized version that preserves the original structure while adding repo-specific guidance.

See [Features — Repo-Aware Step Files](FEATURES.md#repo-aware-step-files-kodysteps) for details.

## Environment Variables

| Variable | Required | Description |
|----------|----------|-------------|
| `ANTHROPIC_API_KEY` | When `provider` is `"anthropic"` (default) | Claude API key (set as GitHub secret) |
| `ANTHROPIC_COMPATIBLE_API_KEY` | When using non-Anthropic provider | Single key for any provider (minimax, openai, google, etc.) |
| `GH_TOKEN` | Auto | Provided by GitHub Actions |
| `LOG_LEVEL` | No | `debug`, `info`, `warn`, `error` |

When `provider` is set to a non-Anthropic value, Kody auto-sets a dummy `ANTHROPIC_API_KEY` (required by Claude Code CLI validation) and routes all API calls through the LiteLLM proxy to your actual provider.

## GitHub Workflow Setup

### Required Secret

```bash
gh secret set ANTHROPIC_API_KEY --repo owner/repo
```

### Required Permission

Settings → Actions → General → **"Allow GitHub Actions to create and approve pull requests"**

### Workflow File

Generated by `init` at `.github/workflows/kody.yml`. Contains 5 jobs:

| Job | Trigger | Purpose |
|-----|---------|---------|
| **parse** | `issue_comment` | Extract command, validate author, parse args |
| **orchestrate** | parse success or `workflow_dispatch` | Run the full pipeline |
| **smoke-test** | `push` to main/dev | Typecheck + CLI loads + dry run |
| **notify-parse-error** | Parse fails | Post error comment |
| **notify-orchestrate-error** | Pipeline fails | Post error comment with logs link |

### Concurrency

```yaml
concurrency:
  group: kody-${{ task_id || issue_number }}
  cancel-in-progress: false
```

One pipeline per task/issue. No parallel runs for the same task.

### Author Validation

Only GitHub collaborators can trigger `@kody`:
- `COLLABORATOR`
- `MEMBER`
- `OWNER`

External contributors are silently ignored.

## Branching Convention

Recommended setup for teams:

```
main (protected — require PR + review)
  └── kody (unprotected — Kody's integration branch)
       ├── 42--add-retry-utility  (per-task, short-lived)
       └── 43--fix-auth-middleware
```

Configure via **Settings → Rules → Rulesets**:

| Branch pattern | Protection |
|---|---|
| `main` | Require PR, require review, require status checks |
| `kody/**` | No protection |

Add your GitHub Actions bot as a **bypass actor** for branch protection.
